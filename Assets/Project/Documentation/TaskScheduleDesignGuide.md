## **Task Type Distribution**

**Programming Tasks:** 10
**Creative Tasks:** 8
**Administrative Tasks:** 7
**Mixed Tasks:** 5

**Total Tasks:** 30 (1 per turn)

## **Difficulty Progression**

**Days 1-2:** Easy tasks, low requirements, generous deadlines
**Days 3-4:** Medium difficulty, mixed skills, moderate deadlines
**Days 5-6:** Higher requirements, sanity management needed
**Days 7-8:** Complex tasks, strategic decisions required
**Days 9-10:** High-stakes tasks, tight deadlines, final push

## **Reward Scaling**

**Early Game (Days 1-3):** 300-500 per task
**Mid Game (Days 4-7):** 500-800 per task
**Late Game (Days 8-10):** 800-1000 per task

**Total Potential Rewards:** ~20,000 (more than enough to reach 5000 target)

## **Implementation Notes**

1. **Task Spawning:** Use TaskDiscoveryManager to spawn tasks according to this schedule
2. **Task Expiry:** Tasks should expire if not completed within Turns Until Expiry
3. **Task Variety:** Ensure each turn has at least one task available
4. **Balancing:** Adjust rewards and requirements based on playtesting
5. **Sanity Pressure:** Gradually increase minimum sanity requirements to force rest management

## **Creating TaskData in Unity Editor**

1. Right-click in Project window → Create → ASAP → TaskData
2. Name the asset (e.g., "FixLoginBug_TaskData")
3. Fill in the fields according to the schedule above
4. Set task type (Programming, Creative, Administrative, Mixed)
5. Save in a dedicated folder (e.g., Assets/Data/Tasks)
